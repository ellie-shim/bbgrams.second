자바스크립트 기본 정리 06 - 객체(Object)

객체 리터럴 (Object Literal)

  • 객체는 한꺼번에 여러 값을 담을 수 있는 통과 같은 자료구조(data structure) 이다.

  • 객체 안에는 이름-값 쌍(name-value pair) 이 저장되는데 이를 객체의 속성(property) 라고 한다.

  1. 속성 이름 부분에 문자열을 사용해도 상관 없다.

  2. '털 색' 과 같이 속성이름에 식별자에 허용되지 않는 문자(한글, 공백)가 들어간 경우 반드시 문자열 표기를 사용해야 한다.

  3. 이미 정의된 변수의 이름을 그대로 속성의 값으로 사용할 수도 있다.(객체 속성 이름과 속성 값으로 넣을 변수의 이름이 같다면(howManyCute) 줄여 쓸 수 있다.)

  4. 대괄호를 사용해서 다른 변수에 저장된 문자열을 그대로 속성의 이름으로 사용할 수 있다.

  5. 객체의 속성값으로 함수를 지정할 수 있다.

  6. 어떤 객체의 속성으로 접근해서 사용하는 함수를 메소드(Method) 라고 한다

  7. 객체 리터럴 안에서 특별한 표기법을 사용해 메소드를 정의할 수도 있다.

  8. this 키워드를 사용하면, 메소드 호출 시에 해당 메소드를 갖고 있는 객체에 접근할 수 있다.

점 표기법, 대괄호 표기법

  1. 속성 접근자(property accessor) 를 이용해 이미 생성된 객체의 속성을 지정해 줄 수 있다.

  2. 식별자에 허용되지 않는 문자(한글, 공백)를 속성 이름에 넣어야 할 경우에는 반드시 대괄호 표기법을 사용해야 한다.

객체 다루기

  • 속성 접근자, delete 연산자, in 연산자 등을 이용해서 객체에 대한 정보를 읽고 쓸 수 있다.

메소드 (Method) - 위 예제

This

  1. 메소드를 사용하면 데이터 와 그 데이터와 관련된 동작 을 객체라는 하나의 단위로 묶어서 다룰 수 있다 . (함수 대신 메소드를 사용하는 핵심적인 이유)

  2. function 키워드를 통해 정의된 함수 내부의 this 키워드는 함수가 호출될 때 결정 된다.

  3. 어떤 객체의 메소드로 사용되느냐에 따라 메소드 내부의 this가 가리키는 객체가 달라질 수 있다

  4. 다만 화살표 함수는 함수가 정의된 문맥에 따라 this가 결정된다 .

프로토타입 (Prototype)

  • 객체 간에 공유되어야 하는 속성과 메소드를 프로토타입이라는 기능을 이용해서 저장한다.

  • 어떤 객체에 프로토타입을 지정하면, 프로토타입의 속성을 해당 객체에서 재사용 할 수 있다.

  1. 프로토타입 기능을 이용해 한 객체에서 다른 객체의 기능을 가져와 사용하는 것을 프로토타입 상속(prototype inheritance) 라고 한다.

  2. petsPrototype은 bibi의 프로토타입이다.

  3. petsPrototype은 doong의 프로토타입이다.

  4. bibi 객체는 petsPrototype 객체를 상속받았다.

  5. doong 객체는 petsPrototype 객체를 상속받았다.

  6. Object.getPrototypeOf 함수를 사용해 어떤 객체의 프로토타입을 읽어올 수 있다.

  7. Object.setPrototypeOf 함수를 사용해 이미 생성된 객체의 프로토타입을 변경할 수 있다. (굉장히 느리므로 사용은 지양한다.)

자바스크립트 기본 정리 04 - 함수 function, 스코프 Scope

  1. 큰 프로그램을 잘게 쪼개어 특정 코드 뭉치를 반복해서 사용할 수 있도록 묶어놓은 코드 뭉치의 단위를 함수라 한다.
  1. 보통 코드가 쓰여진 순서대로 실행되지만, 여러가지 예외가 존재해서 코드의 실행 순서가 이리저리 옮겨다니기도 한다.
  2. 함수 호출도 그 중 하나로, 함수 호출 코드를 만나면 코드의 실행 흐름이 호출된 함수의 내부로 옮겨간다.
  1. 위 코드 실행 흐름
    1. 함수가 정의된다.
    2. 함수를 호출하는 코드를 읽고
    3. 함수 내부의 코드가 실행된다.
    4. 나머지 코드가 실행된다.
  1. 매개변수는 바깥에서 선언된 변수와는 관계없는 독립적인 변수이다.
  1. return 키워드 바로 다음에 오는 값이 함수 호출의 결과값으로 반환되며, 반환되는 즉시 함수 실행이 끝난다. return 뒤에 아무값도 써주지 않거나, 아예 return 구문을 쓰지 않으면 함수는 undefinde 값을 반환한다.

  1. 코드는 일정 범위 안에서만 유효하다.

  2. 특정 변수가 유효한 코드 상의 유효범위를 스코프 Scope 라고 한다.

  3. 매개변수는 함수 스코프를 갖는다.

  4. 함수 내부 코드에서 매개변수 혹은 그 함수 안에서 정의된 변수만 사용 가능한것은 아니고, 바깥 스코프에 정의된 변수도 사용 가능하다.

  5. 식별자와 같은 이름을 찾을때까지 바깥스코프로 올라가는데 이 과정을 스코프 연쇄 scope chain 이라고 한다.

  6. 가장 바깥에 있는 스코프를 최상위 스코프 혹은 전역 스코프 global scope라고 한다.

  • 위와 같이 바깥 쪽 스코프에 존재하는 변수와 같은 이름을 갖는 변수를 안쪽 스코프에서 재정의 할 수 있다. 그렇게 되면 안쪽 스코프에서는 바깥쪽 스코프에 있는 이름들이 무시된다. 이런 현상을 변수가리기 variable shadowing이라고 한다.

  1. 함수도 값이다.

    • 값으로 사용할 수 있는 자바스크립트의 함수는 1급시민이다.
    • 1급시민인 함수를 줄여서 1급함수라 부른다.

  2. 함수를 선언한 뒤 변수에 대입해서 호출할 수 있다.

  3. 배열이나 객체에 넣을 수 있다.

  4. 함수를 다른 함수에 인수로 넘길 수 있다.

  5. 함수에서 함수를 반환할 수 있다.

  1. 함수를 선언할 때는 이름을 붙이지 않아도 되고, 이렇게 이름이 없는 함수를 익명 함수 (anonymous function), 혹은 함수 리터럴 (function literal)이라고 한다.

  2. 익명 함수는 이름이 없기 때문에 이름으로 호출 할 수 없다.

  3. 호출을 하려면 변수에 저장한 후에 변수의 이름을 통해 호출해야 한다.

  4. 익명 함수는 함수를 만든 쪽이 아니라 다른 쪽에서 그 함수를 호출할 때 많이 사용된다.

  5. 대표적인 경우는 함수를 인수로 넘겨줄 때 이다.

  6. 예를들어 filter 메소드에 필터링 할 조건을 표현하는 함수를 넣어주면 filter 메소드쪽에서 각 요소에 대해 함수를 호출한 뒤 true 를 반환한 요소만을 필터링해서 반환한다.

  1. 화살표 함수는 함수 정의를 위한 새로운 표기법이다

  2. 중괄호로 둘러주지 않으면 return구문이 없어도 바로 반환된다.

  3. function 키워드를 통한 함수 정의처럼 여러 구문을 사용하려면 중괄호로 둘러싸주어야한다.
    => 다음 부분을 중괄호로 둘러싸면, 명시적으로 return을 하지 않는 한 아무것도 반환되지 않는다.

  4. 매개변수가 하나밖에 없다면 매개변수 부분의 괄호를 쓰지 않아도 된다.

  5. 화살표 함수는 표기법이 간단하기 때문에 익명 함수를 다른 함수의 인수로 넘길 때 주로 사용된다.

자바스크립트 기본 정리 03 - Null, Undefined

  1. null
  • 없음을 나타내는 값
  • 객체가 없음을 나타낸다.
  • typeof 연산을 해보면 아래와 같은 값을 반환한다.
  1. undefined
  • 없음을 나타내는 값
  • 값이 대입되지 않은 변수 혹은 속성을 사용하려고 하면 undefined를 반환한다.

  1. 프로그래머의 입장에서 명시적으로 부재를 나타내고 싶다면 항상 null을 사용하는 것이 좋다.

  2. 객체를 사용할 때 어떤 속성의 부재를 나타낼 때에는 null보다는, 아예 속성을 정의하지 않는 방식이 간편하므로 더 널리 사용된다.

  1. Null Check

-null이나 undefined는 어떤 변수에도, 어떤 속성에도 들어있을 수 있기 때문에 코드를 짤 때 값이 있는 경우와 값이 없는 경우(즉 null 혹은 undefined인 경우)를 모두 고려해서 짜야한다.

  • 어떤 값이 null 혹은 undefined 인지 확인하는 작업을 null check 라고 한다.
  • 위 코드를 줄일 수 있다
  • null check를 할 때에는 === 보다는 ==를 사용한다
  • == 연산자는 한 쪽 피연산자에 null 혹은 undefined가 오면, 다른 쪽 피연산자에 null 혹은 undefined가 왔을때만 true를 반환하고, 다른 모든 경우에는 false를 반환한다.
